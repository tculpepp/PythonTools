Security Technology Tools I

Review of TCP/IP

To be a good security tester, it is important to have a solid understanding of network and TCP/IP (Transmission control protocol/Internet Protocol). TCP/IP is the main protocol used to connect computers over the Internet. Tools that hackers (both the bad ones and the ethical ones) use run over IP. Since the most widely used IPv4 (standard networking protocol) was developed without security in mind, professionals need the skills and knowledge to fix problems inherent in IPv4.

For people to understand each other, we need to speak in the same language. It is the same when computers communicate with each other over the Internet. This language is called protocol. The mostly commonly used protocol for computers is TCP/IP. TCP/IP is usually referred as TCP/IP stack, which has four layers: network layer, Internet layer, transport layer, and application layer. The network layer moves data across various media; the Internet layer routes data packets via IP addresses; the transport layer manages the flow of data, enclose data segment with a TCP or UDP header. The application layer is the top layer of TCP/IP protocol and includes all the processes that use transport layer protocols to deliver data to the Internet layer. There are many application protocols, such as:

  • Hypertext transfer protocol (HTTP)
  • File transfer protocol (FTP): allows different OSS to transfer files between each other)
  • Simple mail transfer protocol (SMTP): transmits emails across the Internet
  • Secure shell: allows a remote user to log on to a server securely and issue commands
  • Internet relay chat (IRC): enables multiple users to communicate over the Internet in a discussion forum.

In addition, new protocols are frequently being added to the application layer.

COMMON TECHNOLOGY CONTROLS: FIREWALL AND VPN

FIREWALL

According to Webopedia, “a firewall is a system designed to prevent unauthorized access to or from a private network. Firewalls can be implemented in both hardware and software, or a combination of both.”

A firewall is important in building construction; it can prevent a fire from spreading from one part of the building to another. In similar way, a firewall in security obstructs certain information from moving between an untrusted network (the outside world) and the trusted network. There are many variations of firewalls such as a computer system, software that runs on routers, or a separate network. Please read the background information for different categories of firewalls based on their processing mode and structure.

Firewalls play two roles: controlling traffic entering an internal network, and controlling traffic leaving the internal network. Like a security guard, a firewall is installed on a network to protect an organization’s internal network from outside dangers. There are software and hardware firewalls, each with advantages and disadvantages. There are many ways that a hacker can scan the system and exploit vulnerabilities. A firewall adds an extra layer of protection to reduce such threats. There are a few ways that a firewall can achieve this:

Network address translation

If a hacker knows the IP address of a computer, it is easy to scan the system for vulnerabilities and open ports. Hence it is crucial to hide the real IP address from outsiders. The most basic security feature of a firewall is Network Address Translation (NAT), which can map private IP addresses to a public IP address. Thousands of IP addresses can be mapped into one single IP address.

Address lists

Address lists are used to filter traffic based on the source IP address, destination IP address, and ports or services.

Packet filtering

Packet filters screen packets based on information in the packet header: protocol type, IP address, TCP/UDP port.

Stateful packet inspection

Stateful packet filters record session specific information about a network connection and can recognize types of anomalies that most routers ignore.

Application layer inspection

An application-aware firewall not only makes sure that a packet’s source, destination, and port are expected before forwarding the packet, but also if the application protocol is allowed by the rule.

VPN

The virtual private network consortium defines a VPN as "a private data network that makes use of the public telecommunication infrastructure, maintaining privacy through the use of a tunneling protocol and security procedures." A reading on three VPN technologies can be found in background readings. Virtual private networks (VPN) are an implementation of cryptographic technologies that are covered later in this course.

VPNs provide a secure connection between two devices. In the context of VPNs, the term tunnel is misleading; tunnels imply that the link that the tunnel embodies is private, whereas VPNs use virtual tunnels instead, which are set up over the Internet. There are many benefits of setting up a VPN rather than using a leased line: it is cheaper and provides more control over the physical setup.

There are software VPNs and VPN hardware appliances; hardware VPN is able to accommodate more data flow, and software VPNs are less expensive. They also provide for better scalability as the organization or network expands. In addition, users can connect to VPN anywhere in the world with an Internet connection. IPSec is the VPN protocol that is most widely used today.
